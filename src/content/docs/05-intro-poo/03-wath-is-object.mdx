Para resolver problemas con programación orientada a objetos, primero debemos identificar son los objetos que forman parte del problema.

## ¿Qué es un objeto?

Un objeto es una entidad que tiene un estado (atributos) y un comportamiento (métodos).

**Ejemplo:** Un objeto `Coche` puede tener los siguientes atributos y métodos:

- **Atributos:**
    - Marca
    - Modelo
    - Color
    - Velocidad
    - Precio

- **Métodos:**
    - Arrancar
    - Acelerar
    - Frenar
    - Apagar

En este caso, el objeto `Coche` tiene un estado definido por sus atributos (marca, modelo, color, velocidad, precio) y un comportamiento definido por sus métodos (arrancar, acelerar, frenar, apagar).

## Objetos físicos y conceptuales

Los objetos pueden ser físicos o conceptuales:

- **Objetos físicos:** son entidades tangibles que existen en el mundo real, como un coche, una persona, un libro, etc.

- **Objetos conceptuales:** son entidades abstractas que representan conceptos o ideas, como un cliente, una factura, un pedido, cursos, usuarios, etc.

**Ejemplo de un objeto físico:** Un coche.

- **Atributos:**
    - Marca
    - Modelo
    - Color
    - Velocidad
    - Precio

- **Métodos:**
    - Arrancar
    - Acelerar
    - Frenar
    - Apagar

**Ejemplo de un objeto conceptual:** Un usuario.

- **Atributos:**
    - Nombre de usuario
    - Contraseña
    - Email
    - Fecha de registro

- **Métodos:**
    - Registrar
    - Actualizar
    - Eliminar
